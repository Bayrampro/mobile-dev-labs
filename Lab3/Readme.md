## Ответы на контрольные вопросы

**1. Что такое ViewBinding и в чем его преимущество перед findViewById()? Как подключить?**  
`ViewBinding` генерирует типобезопасный класс привязки для каждой XML-разметки и дает прямой доступ к View через поля (`binding.button1`).  
Преимущества: меньше шаблонного кода, нет ручных приведения типов, ниже риск `NullPointerException` и ошибок в id.  
Подключение:
- в `build.gradle.kts` модуля: `buildFeatures { viewBinding = true }`;
- в `Activity`: `binding = ActivityMainBinding.inflate(layoutInflater)` и `setContentView(binding.root)`.

**2. Разница между XML `onClick` и программным `setOnClickListener`. Когда что лучше?**  
`android:onClick` (XML) — декларативно, быстро для простых экранов, но хуже масштабируется и сложнее отлаживать при росте логики.  
`setOnClickListener` (код) — более гибко, удобно для сложной логики, условных обработчиков, переиспользования общего listener.  
Для учебных простых примеров можно XML, для реальных проектов обычно предпочтительнее программный способ.

**3. Что будет, если в XML-обработчике изменить сигнатуру (например, убрать `View v`)? Почему?**  
Приложение упадет во время выполнения при клике (`IllegalStateException`/`InvocationTargetException`), потому что Android ищет методом рефлексии строго метод вида `public fun methodName(view: View)` в Activity/Fragment. При несовпадении сигнатуры метод не находится.

**4. Жизненный цикл Activity. Где лучше инициализировать слушатели и почему?**  
Основные этапы: `onCreate` -> `onStart` -> `onResume` -> `onPause` -> `onStop` -> `onDestroy`.  
Слушатели для кнопок обычно лучше инициализировать в `onCreate`, потому что там создается интерфейс и привязка View; достаточно сделать один раз на весь жизненный цикл Activity.

**5. Что такое анонимный внутренний класс? Как используется для слушателей в Java?**  
Анонимный внутренний класс — это класс без имени, создаваемый сразу в месте использования.  
Для обработчиков в Java обычно пишут:

```java
button.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        // обработка клика
    }
});
```

В Kotlin его роль чаще выполняет лямбда: `button.setOnClickListener { ... }`.

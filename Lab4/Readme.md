## Ход работы

### 1. Структура таблицы

В классе `SqlHelper` создана таблица `students`:

| Поле | Тип SQLite | Назначение |
|------|------------|------------|
| `_id` | `INTEGER PRIMARY KEY AUTOINCREMENT` | Уникальный идентификатор записи |
| `fio` | `TEXT NOT NULL` | ФИО ученика |
| `className` | `TEXT NOT NULL` | Буква класса |
| `classNumber` | `INTEGER` | Номер класса |
| `avgScore` | `REAL` | Средняя оценка |

SQL создания таблицы:

```sql
CREATE TABLE students (
    _id INTEGER PRIMARY KEY AUTOINCREMENT,
    fio TEXT NOT NULL,
    className TEXT NOT NULL,
    classNumber INTEGER,
    avgScore REAL
)
```

---

### 2. Класс `SqlHelper`

`SqlHelper` наследуется от `SQLiteOpenHelper` и отвечает за:
- создание БД и таблицы в `onCreate()`;
- обновление структуры в `onUpgrade()` (текущая реализация: `DROP TABLE IF EXISTS` + повторный `onCreate()`).

Основные константы вынесены в объект `Constants`:
- имя БД: `local_db.db`;
- версия: `1`;
- имя таблицы и названия столбцов.

---

### 3. Модель данных

Создан data class `Student`:

```kotlin
data class Student(
    val id: Int,
    val fio: String,
    val className: String,
    val classNumber: Int,
    val avgScore: Float
)
```

---

### 4. Реализация CRUD-методов

В `SqlHelper` реализованы основные операции:

1. `addStudent(...)`  
Формирует `ContentValues` и добавляет запись через `insert(...)`.

2. `getAllStudents()`  
Выполняет `rawQuery("SELECT * FROM students", null)`, проходит по `Cursor`, собирает список `Student`.

3. `updateStudent(student)`  
Обновляет запись по `_id` через `update(...)`.

4. `deleteStudent(id)`  
Удаляет запись по `_id` через `delete(...)`.

## Ответы на контрольные вопросы

**1. Какие типы данных поддерживает SQLite? Как хранить логические значения и даты?**  
SQLite использует 5 основных типов хранения: `NULL`, `INTEGER`, `REAL`, `TEXT`, `BLOB`.  
Логические значения обычно хранят как `INTEGER` (`0`/`1`).  
Даты можно хранить:
- как `TEXT` (например, `YYYY-MM-DD HH:MM:SS`),
- как `INTEGER` (Unix time),
- как `REAL` (Julian day).

**2. Для чего нужен SQLiteOpenHelper? Назначение onCreate() и onUpgrade().**  
`SQLiteOpenHelper` управляет созданием и версионированием БД.  
`onCreate()` вызывается при первом создании БД и создает таблицы.  
`onUpgrade()` вызывается при увеличении версии БД и выполняет миграцию схемы.

**3. Разница между getWritableDatabase() и getReadableDatabase(). Когда возможна ошибка getWritableDatabase()?**  
`getWritableDatabase()` открывает БД для чтения/записи.  
`getReadableDatabase()` открывает БД только для чтения (или fallback, если запись недоступна).  
Ошибка при `getWritableDatabase()` возможна, например, при нехватке места, проблемах с файловой системой или повреждении БД.

**4. Что такое Cursor? Как перемещаться и зачем закрывать?**  
`Cursor` — это объект результата SQL-запроса.  
Типичный обход: `moveToFirst()` и далее `moveToNext()` в цикле.  
После использования `Cursor` нужно закрывать (`cursor.close()`), чтобы освободить ресурсы и избежать утечек.

**5. Что такое ContentValues и для каких операций применяется?**  
`ContentValues` — контейнер пар `ключ-значение` для столбцов таблицы.  
Используется в `insert()` и `update()` для передачи данных в БД.

**6. Отличие query() и rawQuery(). Пример rawQuery() с плейсхолдером.**  
`query()` — более безопасный и структурированный API для выборки (без ручной сборки SQL-строки).  
`rawQuery()` — выполнение произвольного SQL-запроса вручную.  
Пример:

```kotlin
val cursor = db.rawQuery(
    "SELECT * FROM students WHERE classNumber = ?",
    arrayOf("11")
)
```

**7. Как обновлять структуру существующей таблицы (добавлено новое поле)?**  
Нужно увеличить версию БД (`DATABASE_VERSION`) и реализовать миграцию в `onUpgrade()`.  
Для простого добавления поля можно использовать `ALTER TABLE ... ADD COLUMN ...`.  
Для сложных изменений обычно:
1. создать новую таблицу,
2. перенести данные,
3. удалить старую таблицу,
4. переименовать новую.

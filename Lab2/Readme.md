### 1. Базовое задание — `task_1.xml`

Для первой части использована иерархия вложенных `LinearLayout` с горизонтальной и вертикальной ориентацией.  
Размеры блоков заданы через комбинацию `layout_weight`, `0dp` (по измеряемой оси) и `match_parent`, что позволило получить требуемые пропорции и структуру композиции.

Пример базовой структуры:

```xml
<LinearLayout
    android:orientation="vertical"
    ... >

    <LinearLayout
        android:orientation="horizontal"
        android:layout_weight="1"
        ... >
        ...
    </LinearLayout>
</LinearLayout>
```

Цветные прямоугольники реализованы отдельными блоками (`FrameLayout`) с разными значениями `android:background`.

### 2. Задание по варианту — `task_2.xml`

Во второй части использован контейнер `GridLayout` (`rowCount="4"`, `columnCount="4"`), в котором собрана композиция из 16 элементов, стилизованных как кнопки калькулятора (`0-9`, `+`, `-`, `*`, `/`, `C`, `=`).

Каждый элемент:
- размещается в ячейке сетки с помощью `layout_rowWeight` и `layout_columnWeight`;
- имеет отступы `layout_margin`;
- оформлен через ресурс `@drawable/rectangle`;
- содержит центрированный `TextView` с символом кнопки.

Фрагмент:

```xml
<GridLayout
    android:rowCount="4"
    android:columnCount="4"
    ... >

    <FrameLayout
        android:layout_rowWeight="1"
        android:layout_columnWeight="1"
        android:background="@drawable/rectangle">
        <TextView
            android:text="7"
            android:layout_gravity="center" />
    </FrameLayout>
</GridLayout>
```

## Ответы на контрольные вопросы

**1. Что такое XML? Для каких целей он используется в Android-разработке?**  
XML (`eXtensible Markup Language`) — это язык разметки для хранения и описания структурированных данных. В Android XML используется для описания интерфейсов (`layout`), ресурсов (`strings`, `colors`, `styles`), графики (`drawable`) и конфигураций приложения.

**2. Что такое тег (элемент) в XML? Из каких частей он состоит?**  
Тег (элемент) — базовая единица XML. Обычно состоит из:
- открывающего тега (`<TextView>`),
- атрибутов (`android:layout_width="wrap_content"`),
- содержимого (если есть),
- закрывающего тега (`</TextView>`).  
Для пустых элементов используется краткая форма: `<View ... />`.

**3. Какие менеджеры размещения (контейнеры) вы знаете? Кратко опишите каждый.**  
- `LinearLayout` — размещает элементы в одну линию: вертикально или горизонтально.  
- `RelativeLayout` — располагает элементы относительно родителя и соседних элементов.  
- `ConstraintLayout` — гибкий контейнер с ограничениями (constraints), подходит для сложных экранов.  
- `FrameLayout` — простой контейнер, обычно для наложения элементов друг на друга.  
- `GridLayout` — размещает элементы в таблице (строки/столбцы).  
- `TableLayout` — табличное размещение через строки `TableRow`.

**4. В чём разница между LinearLayout и GridLayout? В каких случаях какой контейнер удобнее использовать?**  
`LinearLayout` удобен для простых последовательных структур (одна ось).  
`GridLayout` удобен, когда нужна сетка из строк и столбцов (клавиатуры, панели кнопок, таблицы элементов).  
Если интерфейс линейный, проще использовать `LinearLayout`; если двумерная структура, удобнее `GridLayout`.

**5. Что такое match_parent и wrap_content? Приведите примеры использования.**  
`match_parent` — элемент занимает весь доступный размер родителя по оси.  
`wrap_content` — элемент занимает размер по своему содержимому.  
Пример: `TextView` с коротким текстом обычно имеет `wrap_content`, а корневой контейнер экрана — `match_parent`.

**6. В чём разница между android:gravity и android:layout_gravity?**  
`android:gravity` выравнивает содержимое внутри самого View (например, текст внутри `TextView`).  
`android:layout_gravity` выравнивает сам View внутри родительского контейнера.

**7. Какие единицы измерения используются в Android? Для чего предназначены dp и sp?**  
Основные единицы: `dp`, `sp`, `px`, `pt`, `mm`, `in`.  
`dp` используется для размеров интерфейса, чтобы сохранять одинаковый визуальный размер на разных плотностях экрана.  
`sp` используется для размера текста и учитывает настройки масштаба шрифта у пользователя.

**8. Как создать простую фигуру (прямоугольник, круг) с помощью XML-ресурса в папке drawable?**  
Нужно создать файл в `res/drawable` с тегом `<shape>` и указать тип фигуры:
- `android:shape="rectangle"` для прямоугольника;
- `android:shape="oval"` для круга/овала.  
Далее задаются параметры через `<solid>`, `<stroke>`, `<corners>`, `<size>`, после чего ресурс применяется как `android:background` или `android:src`.
